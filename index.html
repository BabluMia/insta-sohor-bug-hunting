QNA SECTION

জাভাস্ক্রিপ্ট হলো সিঙ্গেল থ্রেটেড প্রোগ্রামিং ল্যাংগুয়েজ। এর মানে হলো জাভাস্ক্রিপ্ট একসাথে একাধিক কাজ করতে পারে না একটা শেষ করে অন্যটা করে থাকে ।যখন আমরা কোড লিখি শুধু লিখলেই হয় না কোড রান করানো লাগে। এই কোড গুলো ব্রাউজারে রান করানো লাগে। প্রতিটি ব্রাউজারের জাভাস্ক্রিপ্ট এর ইঞ্জিন থাকে যার মাধ্যমে ব্রাউজারে কোড রান হয়। গোগল ক্রোমের জাভাস্ক্রিপ্ট ইঞ্জিনের নাম হলো V8, ফায়ার ফক্সের স্পাইডার মানকি, সাফারির সাফারি। এই জাভাস্ক্রিপ্ট ইন্জিনের মাধ্যমে আমরা আমাদের জাভাস্ক্রিপ্ট কোডের আউটপুট দেখতে পাই।জাভাস্ক্রিপ্ট কোড যখন ব্রাউজারের কাছে আসে তখন Parser কোড Parse করে। এবং AST তে রুপান্তর করে। তারপর জাভাস্ক্রিপ্ট কোড পড়ে পড়ে এক্সিকিউট করে। আর আমরা কোড এর আউটপুট দেখি। অনেক সময় দ্রুত রান করানোর জন্য V8 JIT compiler চালায়। এভাবেই জাভাস্ক্রিপ্ট কাজ করে।

জাভাস্ক্রিপ্ট SYNCHRONOUS,ASYNCHRONOUS এই দুই ভাবে কাজ করে থাকে। যখন অনেক গুলো function কল করা হয় আর সেখানে কিছু যদি API আর setTimeout() থাকে যেগুলো নরমাল টাইম থেকে একটু বেশি সময় নেয় লোড হতে তখন মুলত EVENT LOOP এর কাজ শুরু। ENGNE এ CALL STACK এর মাধ্যমে Function এক্সিকিউট হয়। তাই CALL STACK এ প্রথমে একটা main() কল হয় তারপর যে function আগে আছে তা CALL STACK এ আগে উঠে আসে। এখন function গুলোর মধ্যে যে setTimeout(), API থাকে এই গুলো রান হতে টাইম লাগে তাই এই গুলো WEB API/ C++ API এর মধ্যে চলে যায় এবং ঐখানে থাকে লোড হওয়ার আগ পর্যন্ত। তারপর চলে আসে CALLBACK QUEUE তে। এখন এই CALLBACK QUEUE থেকে CALL STACK এ উঠানোর মাধ্যম ই হলো জাভাস্ক্রিপ্টের EVENT LOOP। CALLBACK QUEUE তে যে আগে আসে তাকে CALL STACK এ আগে উঠানো হয় যা EVENT LOOP এর মাধ্যমে হয়ে থাকে।

Local Storage এর সাইজ 10 MB হয়ে থাকে যদিও অনেক যায়গায় 5 MB বলা আছে। আর Session Storage এর সাইজ হলো 5 MB।
Local Storage এর ডাটা নিজে ডিলট করতে হয়। আর Session Storage এর ডাটা Tab বন্ধ করলে চলে যায়।
Local Storage ব্রাউজার এর নিদিষ্ট কোনো ট্যাবের জন্য নয়। আর Session Storage নিদিষ্ট ট্যাবের জন্য, ট্যাব অফ করলে ডাটা মুছে যায়
Local Storage এর ডাটার মেয়াদ থাকো ১-২ বছর পর্যন্ত যেখানে Session Storage এর কোনো মেয়াদ থাকে না Tab বন্ধ করলে চলে যায় ।
